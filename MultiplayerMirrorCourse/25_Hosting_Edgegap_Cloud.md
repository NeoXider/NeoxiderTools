## Цели

Понять слой между «у меня есть билд» и «в мире есть живой сервер матча»: куда смотреть в примерах Mirror для актуального **Edgegap** plugin, как описать **контракт** между оркестратором и процессом игры, как не разориться на «забытых» инстансах.

---

## Проблема и контекст

| Ручной SSH на каждый матч | Операционный ад, нет автоскейла |
|---------------------------|----------------------------------|
| «Подняли сервер и забыли» | Счёт в облаке растёт неделями |
| Нет health-check | Платёжные игроки попадают на мёртвый порт |

---

## Теория

**Оркестратор** (Edgegap, k8s job, облачная функция + VM) поднимает инстанс, передаёт аргументы (`-port`, `-matchId`, `-region`), проверяет health, списывает инстанс по таймауту или завершению матча. Mirror остаётся **внутри процесса игры**; облако решает **где** он бежит и **как долго**.

Важно разделять:

| Слой | Ответственность |
|------|------------------|
| Оркестратор | Жизненный цикл VM/контейнера, сеть, DNS |
| Ваш server binary | `NetworkServer`, транспорт, игровая логика |
| Лаунчер клиента | Получил IP:порт / relay token, подключился |

---

## Практика

1. Прочитайте README примеров Mirror для Edgegap на **вашей** версии пакета — URL и шаги меняются.
2. Составьте список **env vars / CLI**, которые ваш сервер уже читает (урок 14).
3. Нарисуйте временную диаграмму: «запрос матча» → «инстанс готов» → «клиенты коннектятся».

### Шаг 1 — контракт запуска

Документируйте: минимальная ОС, открытые порты, нужен ли **outbound** HTTPS для бекенда (часто да). Без этого DevOps будет гадать.

### Шаг 2 — graceful shutdown

При SIGTERM процесс должен: перестать принимать новых (`listen` false в терминах v96, см. урок 1), сохранить матч, закрыть соединения, выйти. Оркестратор даёт ограниченное время — проверьте его на стейдже.

### Шаг 3 — регионы

Первый этап — **один** регион. Мультирегион добавляет задержку к БД и рассинхрон часов; подключайте, когда CCU стабильно требует.

---

## Типичные ошибки

- Забыть закрыть порт / инстанс — счёт в облаке растёт.
- Путать **игровой UDP** порт и **HTTP health** — security group открыт не туда.
- Хардкод секретов в образе Docker — используйте runtime env из секретов.
