# UIPage

**Назначение:** компонент UI-страницы для работы с `PM` (Page Manager). Хранит `PageId`, режим popup и настройки совместимости для поведения открытия/закрытия.

## Подключение

1. Добавьте `UIPage` на корневой GameObject страницы.
2. Назначьте `Page Id`.
3. Если страница должна открываться поверх других, включите `Popup`.
4. Если страница не должна закрываться при эксклюзивном переключении, включите `Ignore On Exclusive Change`.
5. Для кастомной анимации добавьте отдельный проектный компонент рядом с `UIPage`.

## Основные поля (Inspector)

| Поле | Описание |
|------|----------|
| `Page Id` | Идентификатор страницы для переключения через `PM`. |
| `Popup` | Страница открывается поверх текущих страниц без их деактивации. |
| `Ignore On Exclusive Change` | `PM` не деактивирует эту страницу при обычном эксклюзивном переключении. |
| `Animation Mode` | Compatibility-настройка для старых сцен и будущих проектных расширений. Базовый sample не требует tween-компонентов. |

## Animation Mode

| Режим | Поведение |
|------|-----------|
| `ForwardOnly` | Совместимость со старыми настройками. В базовом sample страница включается/выключается сразу. |
| `BackwardOnly` | Совместимость со старыми настройками. В базовом sample страница включается/выключается сразу. |
| `ForwardAndBackward` | Совместимость со старыми настройками. В базовом sample страница включается/выключается сразу. |

При эксклюзивном переключении через `PM` (например Menu -> Shop) входящая страница включается, затем исходящие страницы закрываются через `EndActive()`. Страницы с `Ignore On Exclusive Change` не трогаются. Активные `Popup`-страницы по умолчанию закрываются при открытии обычной non-popup страницы (`PM.closePopupsOnExclusivePageChange = true`). Если popup должен переживать exclusive-переходы, выключите эту настройку в `PM`. Открытие popup по-прежнему идет через `ChangePage` -> `ActivePage` без выключения фоновой страницы.

## API

| Метод | Назначение |
|------|------------|
| `StartActive()` | Включить страницу. |
| `EndActive()` | Закрыть страницу. Если объект уже неактивен в иерархии — только `SetActive(false)`. |
| `SetActive(bool)` | Напрямую включить/выключить GameObject страницы. |

## Совместимость

Старые поля `_playBackward` и `_onlyPlayBackward` автоматически мигрируют в `Animation Mode`:

- `_onlyPlayBackward = true` → `BackwardOnly`;
- `_playBackward = true` → `ForwardAndBackward`;
- `_playBackward = false` → `ForwardOnly`.
