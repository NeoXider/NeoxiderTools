## Цели

Проверять действия игрока на сервере, не доверять клиентским числам, проектировать урон и позицию из единого источника правды.

---

## Проблема и контекст

| Доверие клиенту | Итог |
|-----------------|------|
| Клиент шлёт «убил босса» | Чит |
| Клиент задаёт урон в `Command` без проверки | One-shot эксплойт |
| Client authority на движении без лимитов | Speedhack / телепорт |

---

## Теория

**Золотое правило:** состояние, от которого зависит честность (HP, инвентарь, матч), меняется **только** на сервере. Клиент либо запрашивает (`Command`), либо отображает (`SyncVar`/Rpc).

Попадания: сервер делает **raycast** из санитизированной позиции/направления с лимитами скорости стрельбы и дистанции.

---

## Практика

```csharp
[Command]
void CmdMeleeAttack()
{
    if (Time.time < nextMeleeAllowed) return;
    nextMeleeAllowed = Time.time + meleeCooldown;
    if (Physics.Raycast(meleeOrigin.position, meleeOrigin.forward, out var hit, 2f))
    {
        var hp = hit.collider.GetComponent<Health>();
        if (hp != null) hp.ServerApplyDamage(10);
    }
}
```

---

## Типичные ошибки

- Проверка дистанции без учёта пинга в стрельбе — см. урок 10.
- Вся экономика в `ClientRpc` — перенос на сервер.

---

## Советы и паттерны

- Серверные **таймстемпы** и кулдауны на соединение.
- Логируйте подозрительные `Command` (частота, параметры).
