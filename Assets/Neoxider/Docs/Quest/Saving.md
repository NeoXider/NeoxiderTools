# Сохранение прогресса квестов

Опциональная интеграция: сохранение и загрузка состояний квестов (QuestState) через вашу систему сейвов. Модуль Quest не содержит готовой привязки к Save — её можно добавить в игре.

---

## Что сохранять

- Список **QuestState**: для каждого квеста — **QuestId**, **Status**, прогресс по целям (массив чисел и флаги «цель выполнена»).
- QuestState сериализуем ([Serializable], поля с [SerializeField]): его можно сериализовать в JSON или бинарный формат и хранить в составе общих данных игрока.

---

## Варианты реализации

1. **SaveField / ISaveableComponent (Neoxider Save)**  
   На QuestManager (или отдельный компонент-обёртка) реализовать интерфейс сохранения и записывать список состояний `List<QuestState>`. При загрузке — очистить текущий реестр в QuestManager и восстановить состояния из сейва. Для этого в QuestManager нужен метод вида `LoadStates(List<QuestState> loaded)` или возможность инициализации из внешнего источника (если такого API нет — его добавляют в свой форк или обёртку).

2. **Свой менеджер сейвов**  
   При сохранении: обойти `QuestManager.Instance.AllQuests`, сериализовать каждое QuestState в DTO (id, status, progress[], completed[]). При загрузке: десериализовать и передать данные в QuestManager для восстановления списка _states.

3. **Сохранение только Id завершённых квестов**  
   Если достаточно «какие квесты пройдены», сохранять только список questId со статусом Completed. При загрузке создавать QuestState с Status = Completed и пустым прогрессом (или не создавать активные — в зависимости от геймдизайна).

---

## Загрузка при старте сцены

- В **Awake** или **Start** после инициализации системы сохранений вызвать загрузку данных квестов.
- Восстановить список QuestState в QuestManager (через публичный или внутренний API, в зависимости от выбранного варианта).
- Подписки на события (OnQuestCompleted и т.д.) делаются как обычно — после загрузки новые события будут срабатывать корректно.
