# Урок 12: Сетевой UI

---

**Навигация:** [Оглавление](README.md) · [Оформление](LESSON_STYLE.md) · **Базовый трек** · урок **12/15** · Mirror **v96.x**

|----------------|------------------------------------------------------------------|

---

## Цели

Отделить `NetworkBehaviour` от UI, избежать `Find` в `Update`, корректно показывать ошибки при разрыве соединения.

---

## Проблема и контекст

| Смешение UI и сети | Результат |
|--------------------|-----------|
| `GameObject.Find` каждый кадр | Лаги, хрупкость |
| HUD читает чужие `SyncVar` без фильтра | Неверные полоски HP |

---

## Теория

- **World Space** UI — на объекте в мире (имена над головой).
- **Overlay HUD** — один канвас; показывает данные **локального** игрока через подписку на его модель.

Сетевой слой кидает события (`OnHealthChanged`), View обновляет `TMP_Text`.

При Interest Management + видимости учитывайте **Canvas** на объектах вне зоны: с **`v96.10.0`** `SetHostVisibility` затрагивает **все Canvas renderers** ([CHANGELOG](CHANGELOG_Course_Mirror96.md)).

---

## Практика

1. `PlayerStats` (`NetworkBehaviour`) с `SyncVar` + hook → C# event.
2. `HudBinder` слушает event только для `isLocalPlayer` экземпляра.

---

## Типичные ошибки

- Две `EventSystem` в additive сцене — отключайте дубликаты.
- Нет экрана «соединение потеряно» — игрок зависает (см. урок 15).

---

## Советы и паттерны

- `CanvasGroup` для блокировки ввода при лоадинге между сценами.

---

## Домашнее задание

Сделайте панель ошибки с кнопкой «в меню», вызываемую из кастомного `NetworkManager.OnClientDisconnect`. Добавьте **причину** (таймаут / kick / версия) через параметр или `NetworkManager` поле, заполняемое до вызова UI.
