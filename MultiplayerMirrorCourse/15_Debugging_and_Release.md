## Цели

Воспроизводить плохую сеть в редакторе, обрабатывать `Disconnect`, пройти чек-лист перед стором.

---

## Проблема и контекст

| Только localhost | Риск |
|------------------|------|
| «У меня работает» | Релиз ломается на 150 ms + 3% loss |
| Нет UX при обрыве | Зависший HUD, «призраки» в мире |

---

## Теория

**Latency / jitter / packet loss** — норма интернета. KCP смягчает часть проблем reliable-каналов, но джиттер всё равно влияет на `NetworkTransform` — настраивайте **`SnapShotSettings`** (актуальный путь в v96 вместо устаревших полей в старых туториалах).

`NetworkManager` / Transport: **`OnApplicationQuit` deprecated** → переносите очистку в **`OnDestroy`**.

**v96.0.1:** **`NetworkManager.StopClient`** больше **не** вызывает `OnClientDisconnectInternal` напрямую — цепочку разрыва инициирует **транспорт**; при кастомной обработке отключения проверьте подписки на события транспорта и `OnClientDisconnect` в менеджере.

**KCP:** при завершении приложения обработка **`OnClientError`** стала устойчивее к `null` — меньше шума в логах при shutdown.

---

## Практика

1. Включите симулятор задержки на транспорте (см. документацию вашей версии KCP / Mirror).
2. Реализуйте `ShowDisconnectReason(string)` из `NetworkManager.OnClientDisconnect`.
3. Пройдите чек-лист:

- [ ] Нет доверенных клиентским `SyncVar` для боя
- [ ] Смена сцен только через сетевой API
- [ ] IM включён для больших карт
- [ ] Симуляция 150 ms не делает игру неиграбельной

---

## Типичные ошибки

- Тест без **двух клиентов** одновременно.
- Логи только `Debug.Log` без уровней на сервере.

---

## Советы и паттерны

- Mirror → **Diagnostic Logging** в меню для поиска спамящих `NetworkIdentity`.
- ParrelSync / второй билд — обязательный рабочий процесс.
