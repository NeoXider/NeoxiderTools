## Цели

Использовать сетевые коллекции для инвентаря и таблиц прогресса, понимать дельта-операции, знать про поддержку **`HashSet`** в RPC/SyncVar/сообщениях в v96.

---

## Проблема и контекст

| Подход | Проблема |
|--------|----------|
| `[SyncVar] List<T>` целиком | Полная пересылка списка при любом изменении |
| Ручная строка JSON в `SyncVar` | Размер, парсинг, без типобезопасности |

`SyncList` / `SyncDictionary` шлют **операции** (add/remove/change), а не весь массив.

---

## Теория

- **`SyncList<T>`** — порядок важен (инвентарь).
- **`SyncDictionary<K,V>`** — быстрый доступ по ключу (квесты).
- **`SyncHashSet<T>`** — уникальные id (ачивки).
- В **v96** тип **`HashSet`** поддержан также в параметрах Command/Rpc и `NetworkMessage` — проверьте лимиты размера пакета.

Кастомные типы — через `ReadWriteExtensions` (сериализаторы).

**v96.0.1:** генератор **Reader/Writer** корректно работает **между сборками (asmdef)** — выносите сетевые DTO в отдельную сборку при необходимости.

**96.9.21+:** для стабильных 16-битных хешей используйте **`GetStableHashCode16`** вместо грубого усечения — меньше коллизий имён/типов в пользовательских контрактах.

---

## Практика

```csharp
readonly SyncList<ItemStack> inventory = new SyncList<ItemStack>();

public override void OnStartServer()
{
    inventory.Add(new ItemStack { id = 1, count = 3 });
}
```

Подпишитесь на `inventory.Callback` на клиенте для UI.

---

## Типичные ошибки

- Менять коллекцию с клиента — только сервер.
- Хранить в элементах мутабельные ссылки на несетевые Unity-объекты.

## Советы и паттерны

- Держите элементы списка **как struct** или неизменяемые snapshot-структуры.
- Большие каталоги предметов — по **id** + SO на клиенте (урок 11).

## Углублённо: дельта-операции

`SyncList` и родственники шлют **операции** (добавление, удаление, изменение по индексу), а не целиком коллекцию. Это критично для инвентаря из 40 слотов: одно перемещение — одна операция, а не 40 полей.

### Выбор структуры

| Структура | Когда |
|-----------|--------|
| `SyncList` | Порядок важен (инвентарь в ряд) |
| `SyncDictionary` | Квесты, модификаторы по id |
| `SyncHashSet` | Уникальные открытые зоны / ачивки |

На клиенте подпишитесь на `inventory.Callback` и обновляйте только затронутые слоты. Не пересобирайте весь Scroll View без нужды — GC и фризы.

## Сериализация кастомных struct

Держите элементы **небольшими** и явно сериализуемыми. Для сложных типов — `ReadWriteExtensions` и тест на круговую сериализацию в Editor.

## FAQ

**Вопрос:** Можно ли хранить `Sprite` в элементе списка?  
**Ответ:** Нет как сетевые данные; храните id, клиент мапит на SO (урок 11).

**Вопрос:** Как откатить ошибочное добавление на сервере?  
**Ответ:** `RemoveAt` / очистка с валидацией; не мутируйте структуру за предами API списка.
