# Компонент PlayAudio

## 1. Введение

`PlayAudio` — это простой компонент-триггер, который позволяет воспроизвести звуковой эффект из глобального `AM` (Audio Manager). Его можно разместить на любом объекте, и он сможет проиграть звук либо при старте сцены, либо по вызову публичного метода.

Компонент поддерживает два режима работы:
- **Legacy режим**: Воспроизведение по ID из массива звуков в AM (обратная совместимость).
- **Новый режим**: Воспроизведение напрямую из списка `AudioClip` с возможностью случайного выбора.

Это удобная утилита, чтобы быстро озвучить какое-либо событие в игре (появление объекта, взрыв и т.д.), не написав ни строчки кода.

## 2. Описание класса

### PlayAudio
- **Пространство имен**: `Neo.Audio`
- **Путь к файлу**: `Assets/Neoxider/Scripts/Audio/AudioSimple/PlayAudio.cs`

**Описание**
Воспроизводит звук из `AM` по указанному `id` или из списка клипов с возможностью случайного выбора.

**Ключевые особенности**
- **Двойной режим работы**: Поддерживает как воспроизведение по ID (legacy), так и напрямую по `AudioClip`.
- **Случайный выбор**: Если задан список клипов и включен `useRandomClip`, выбирается случайный клип из списка.
- **Интеграция с `AM`**: Напрямую вызывает `AM.I.Play()`.
- **Настройка в инспекторе**: Позволяет выбрать `id` клипа (legacy) или список клипов и настройки.
- **Воспроизведение на старте**: Опция `_playOnAwake` позволяет проиграть звук сразу после создания объекта.

**Поля в инспекторе**
- `_clipType` (`int`): ID звука в массиве AM (legacy режим).
- `_clips` (`AudioClip[]`): Список клипов для воспроизведения (новый режим).
- `_useRandomClip` (`bool`): Если `true` и в списке больше 1 клипа - выбирается случайный.
- `_playOnAwake` (`bool`): Если `true`, звук проигрывается автоматически при старте.
- `_volume` (`float`): Громкость воспроизведения (0-1).

**Публичные методы**
- `AudioPlay()`: Воспроизводит настроенный звуковой эффект. Если заданы клипы - использует их (с случайным выбором если включено), иначе использует режим по ID.
