## Цели

Свести к минимуму уязвимости на уровне соединений и команд: лимиты, таймауты, разделение игрового порта и HTTPS; понимать, **что** даёт шифрование транспорта Mirror и **чего** оно не даёт.

---

## Проблема и контекст

| Открытый порт без лимитов | Утопление `Command` флудом |
|---------------------------|----------------------------|
| Секреты в клиенте | Любой декомпилятор = компрометация |
| Один порт на всё | Нельзя отключить админку без игры |

---

## Теория

Защита **слоями**: облачный WAF / security group → приложение (max msg rate на соединение) → игровая логика (кулдауны, дистанция). Секреты только на сервере и в CI secrets.

**Транспорт Mirror `Encryption`** (см. [урок 1](01_Mirror_Architecture.md), [CHANGELOG](CHANGELOG_Course_Mirror96.md)): отдельный TCP-слой с шифрованием канала между клиентом и выделенным сервером. В **v96** реализует **`PortTransport`**; не путать с **HTTPS** вашего REST API для логина/магазина.

> **Важно:** шифрование канала не заменяет **валидацию** `Command` — злоумышленник с легитимным клиентом всё равно может слать плохие команды.

---

## Практика

1. Таблица: каждый `Command` → max частота → действие при превышении (disconnect / ignore / flag account).
2. Таймаут аутентификации (урок 19).
3. Отдельный процесс или порт для админки только по VPN.

### Шаг 1 — модель угроз на салфетке

Запишите 5 угроз: чит клиента, флуд, подмена пакета, утечка токена, инсайдер. Для каждой — один контрмерный шаг **реалистичный** для вашей стадии.

### Шаг 2 — rate limit

Реализуйте счётчики на соединение: скользящее окно по времени. Лимиты должны быть **разными** для разных команд (прыжок vs чат).

### Шаг 3 — наблюдаемость

Логируйте срабатывания лимитов агрегировано — иначе вы не увидите начало атаки.

## Типичные ошибки

- Хранить Steam / backend ключ в репозитории.
- Доверять `isClient` данным для экономики без сервера.
- Публиковать RCON в открытый интернет без ACL.

---

## Советы и паттерны

Отдельный **админ-порт** или RCON только по VPN, не в открытый интернет без нужды.

Используйте **разные ключи** для dev/staging/prod.
