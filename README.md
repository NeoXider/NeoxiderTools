## Обзор

Ассеты **Neoxider Audio, Bonus & Additional Systems** объединяют в себе ряд независимых, но интегрируемых компонентов для управления звуком, бонусными механиками и дополнительными системами UI. Главные возможности набора:
- **Аудио модуль:** централизованное управление звуком, эффекты, музыка, а также динамическое обновление аудио-интерфейса.
- **Бонусная система:** реализация коллекции бонусов, вращающегося колеса для случайных наград и системы наград за время.
- **Дополнительные UI модули:** демонстрация фальшивого лидерборда, управление отображением времени и игровой валюты.

## Аудио модуль

### AM (Audio Manager)

**Файл:** `Scripts/Audio/AudioSimple/AM.cs`  
**Описание:**  
Синглтон-менеджер аудио, контролирующий воспроизведение звуковых эффектов и музыки. Обеспечивает:
- Проигрывание аудио-клипов по идентификатору;
- Настройку громкости для эффектов и музыки;
- Автоматическое создание и валидацию AudioSource для музыки.

**Пример использования:**
```csharp
// Проигрывание звукового эффекта
AM.Instance.Play(0);

// Запуск музыки
AM.Instance.PlayMusic(1);
```

---

### PlayAudio и PlayAudioBtn

**Файлы:**  
- `Scripts/Audio/AudioSimple/PlayAudio.cs`  
- `Scripts/Audio/AudioSimple/PlayAudioBtn.cs`

**Описание:**  
Эти скрипты выступают в роли помощников для воспроизведения аудио:
- **PlayAudio:** запускает воспроизведение при активации объекта или на старте.
- **PlayAudioBtn:** привязывает аудио к событию нажатия по UI кнопке.

**Примечание:**  
Оба компонента используют `AM` для проигрывания заданных клипов.

---

### RandomMusicPlayer

**Файл:** `Scripts/Audio/RandomMusicPlayer.cs`  
**Описание:**  
Компонент для случайного воспроизведения музыкальных треков. При отсутствии проигрываемого трека выбирает новый, избегая повторов при наличии нескольких вариантов.

---

### SettingMixer

**Файл:** `Scripts/Audio/SettingMixer.cs`  
**Описание:**  
Обеспечивает возможность управления громкостью посредством AudioMixer. Изменяет параметр "Master" в зависимости от настроек, переданных через UI (например, слайдер).

---

### SettingsAudio и AMSettingChange

#### SettingsAudio

**Файл:** `Scripts/Audio/SettingsAudio.cs`  
**Описание:**  
Управляет настройками аудио (громкость и mute) для звуковых эффектов и музыки. Сохраняет начальные значения и использует `PlayerPrefs` для постоянства настроек между сессиями.

#### AMSettingChange

**Файл:** `Scripts/Audio/View/AMSettingChange.cs`  
**Описание:**  
Связывает UI элементы (Toggle, Slider) с методами `SettingsAudio`, обеспечивая динамическую настройку аудио параметров в реальном времени.

---

### UI Audio Extension Level

**Файл:** `Scripts/Audio/UIAudioExtensionLevel.cs`  
**Описание:**  
Эта компонента расширяет функциональность аудио-интерфейса, предоставляя:
- Динамическое отображение уровня аудио;
- Интеграцию с другими UI-элементами для визуальной индикации громкости;
- Возможность анимированного обновления индикаторов при изменении уровней звука.

**Пример использования:**  
При изменении уровня звука с помощью слайдера, соответствующий индикатор обновляется, визуально отражая текущее значение громкости.

## Бонусная система и игровые награды

### Коллекция (Collection)

**Файл:** `Scripts/Bonus/Collection/Collection.cs`  
**Описание:**  
Компонент для управления коллекционными бонусами. Отслеживает состояние разблокированных элементов, сохраняет данные через `PlayerPrefs` и активирует события при получении нового предмета.

**Особенности:**
- Проверка уникальности полученных бонусов;
- События для обновления UI коллекции.

---

### Колесо фортуны (Wheel of Fortune)

**Файл:** `Scripts/Bonus/WheelOfFortune.cs`  
**Описание:**  
Модуль реализует вращающееся колесо с наградами:
- Обеспечивает интерактивную анимацию с постепенным замедлением;
- Поддерживает настройку списка наград, длительности вращения и параметров анимации;
- По завершению вращения фиксирует выбранную награду и инициирует соответствующее событие.

**Пример использования:**  
При активации колеса вызывается метод `Spin()`, который запускает анимацию. По окончании вращения, выбранная награда передается в систему бонусов или обрабатывается отдельно.

---

### Вознаграждение по времени (Time Reward)

**Файл:** `Scripts/Bonus/TimeRewardManager.cs`  
**Описание:**  
Компонент для выдачи бонусов на основе времени. Ключевые особенности:
- Настройка интервалов начисления бонусов (например, каждые 30 минут);
- Отслеживание времени последнего получения награды с использованием `PlayerPrefs`;
- Обновление UI: отображение обратного отсчёта до следующей награды;
- Интеграция с другими системами (например, валютой или бонусной коллекцией).

**Пример использования:**  
Метод `CheckRewardAvailability()` проверяет, прошёл ли нужный интервал, и если да – активирует бонус, обновляя интерфейс и данные игрока.

## Дополнительные UI модули

### Fake LeaderBoard

**Файл:** `Scripts/UI/FakeLeaderBoard.cs`  
**Описание:**  
Модуль для отображения «фальшивого» лидерборда:
- Имитирует таблицу лидеров для повышения вовлечённости игрока;
- Поддерживает настройку шаблонных данных (имена, очки, позиции);
- Обновляет UI в реальном времени или по запросу.

**Пример использования:**  
При запуске игры или по событию, система заполняет таблицу данными из шаблона, создавая иллюзию активного сообщества игроков.

---

### Time

**Файл:** `Scripts/UI/TimeManager.cs`  
**Описание:**  
Модуль для управления и отображения игрового времени:
- Обеспечивает функции обратного отсчёта для различных событий;
- Отображает текущее время игрового сеанса или системное время;
- Может синхронизироваться с наградами по времени и другими элементами UI.

**Пример использования:**  
Таймер начинает отсчёт при запуске уровня, а UI-элемент обновляется каждую секунду, показывая оставшееся время до события.

---

### Money

**Файл:** `Scripts/UI/MoneyManager.cs`  
**Описание:**  
Компонент для системы игровой валюты:
- Отображает текущий баланс игрока;
- Обеспечивает функции начисления и списания средств;
- Интегрируется с другими игровыми системами, такими как бонусы и покупки.

**Пример использования:**  
При получении награды через Wheel of Fortune или Time Reward, сумма валюты обновляется, а UI-индикатор мгновенно отражает новый баланс.

## Расширения и утилиты

### CoroutineExtensions

**Файл:** `Scripts/Extension/CoroutineExtensions.cs`  
**Описание:**  
Набор методов-расширений для упрощения работы с корутинами в Unity:
- `WaitAndExecute` – выполнение действия по истечении задержки;
- `WaitUntilAndExecute` / `WaitWhileAndExecute` – ожидание выполнения условия;
- `ExecuteAfterFrames`, `ExecuteNextFrame` и `ExecuteAtEndOfFrame` – исполнение действий с привязкой ко времени кадров.

**Пример использования:**
```csharp
// Запустим корутину, которая выполнит действие через 2 секунды:
StartCoroutine(this.WaitAndExecute(2f, () => Debug.Log("Выполнено через 2 секунды")));
```

## Зависимости и требования

- **Unity:** Рекомендуется версия 2019 и выше.
- **DOTween:** Для анимации (например, в «Колесе фортуны» и «Box») требуется установить DOTween.
- **PlayerPrefs:** Используется для сохранения настроек и прогресса бонусной системы.
- **Пользовательские атрибуты:** Некоторые компоненты могут применять атрибуты вроде `[GetComponent]` или `[FindAllInScene]` для упрощения настройки – убедитесь, что соответствующие утилиты включены в проект.

## Интеграция ассетов Neoxider через Unity Package Manager

Если вы хотите интегрировать ассеты **Neoxider Audio, Bonus & Additional Systems** в ваш Unity-проект, импортируя только содержимое папки `/Assets/Neoxider`, используйте следующий Git URL:

```
https://github.com/NeoXider/NeoxiderTools.git?path=/Assets/Neoxider
```

При добавлении этого URL в Unity Package Manager (в файле `manifest.json` или через интерфейс Unity) менеджер пакетов загрузит и установит только ту часть репозитория, которая располагается в папке `/Assets/Neoxider`.

Примечание: данный механизм поддерживается Unity Package Manager и отличается от стандартных возможностей Git, позволяя импортировать только нужную часть репозитория.
