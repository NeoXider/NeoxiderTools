## Теория

Steam даёт **lobby** и доставку до клиентов; Mirror всё равно нужен **NetworkManager** и ваша server-authoritative логика. Транспорт подменяет «как байты доходят» — API игры остаётся Mirror.

Для **WebGL** следите за **SimpleWebTransport**: на линии **96.9.x** исправлены аллокации, совместимость **WebAssembly 2023** и залив консоли ошибками при unreliable **EntityStateMessages** — при билде WebGL держите пакет обновлённым ([CHANGELOG](CHANGELOG_Course_Mirror96.md)).

## Практика

1. Найдите в документации актуальный Steam-транспорт для вашей версии Mirror.
2. Прототип: lobby → старт host → join friend.

---

## Типичные ошибки

- Инициализировать Steam после `NetworkManager.Start` без порядка — гонки.
