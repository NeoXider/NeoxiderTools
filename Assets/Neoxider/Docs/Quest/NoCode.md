# Квесты без кода (NoCode)

Настройка квестов через инспектор и UnityEvent: принятие квеста по кнопке, зачёт цели по условию (NeoCondition), показ UI при завершении.

---

## Сценарий 1: Принять квест по кнопке

1. В сцене есть **QuestManager** (добавлен на GameObject). В **Known Quests** перечислены нужные QuestConfig. В **Condition Context** назначен объект для проверки условий (например, игрок).
2. На кнопку (или панель с кнопкой) добавьте компонент **Quest Accept Trigger** (Neoxider → Quest → Quest Accept Trigger).
3. В инспекторе Quest Accept Trigger укажите поле **Quest** — перетащите ваш QuestConfig.
4. У кнопки в **Button → On Click ()** добавьте вызов: объект с QuestAcceptTrigger → функция **QuestAcceptTrigger.AcceptQuest** (без параметров).

При нажатии кнопки квест будет принят, если выполнены условия старта из конфига.

---

## Сценарий 2: Цель по условию (NeoCondition → зачёт цели)

1. Создайте GameObject (или используйте существующий), на котором будет проверяться условие. Добавьте компонент **NeoCondition** (Neoxider → Condition → NeoCondition).
2. В NeoCondition настройте **Conditions** (как в [документации NeoCondition](../Condition/NeoCondition.md)) — например «здоровье игрока >= 50» или «переменная счётчика >= 1».
3. На тот же объект (или дочерний) добавьте **Quest Objective Notifier** (Neoxider → Quest → Quest Objective Notifier). В полях укажите **Quest** (ваш QuestConfig) и **Objective Index** (индекс цели: 0, 1, 2, …).
4. В NeoCondition в блоке **Events** → **On True** добавьте вызов: объект с QuestObjectiveNotifier → **QuestObjectiveNotifier.NotifyComplete**.

Когда условие NeoCondition станет true, вызовется NotifyComplete → QuestManager засчитает указанную цель. Для цели типа CustomCondition/ReachPoint/Talk этого достаточно (цель выполняется один раз).

---

## Сценарий 3: Показать UI при завершении квеста

1. У **QuestManager** в инспекторе найдите блок **Events** → **On Quest Completed** (UnityEvent&lt;string&gt;).
2. Добавьте вызов: перетащите ваш UI-объект (панель, попап) и выберите функцию, которая показывает панель или воспроизводит анимацию. В событие передаётся **questId** (string) — при необходимости можно использовать его для разного поведения по квесту (в NoCode без скрипта доступен только один общий обработчик на все квесты).

Если нужна разная реакция на завершение разных квестов без кода — можно сделать несколько скрытых панелей и в одном скрипте-мосте по questId включать нужную панель, подписавшись на On Quest Completed из кода (см. [Code](Code.md)).

---

## Ограничения NoCode

- **Сложная логика награды** (разные награды в зависимости от выбора игрока или состояния мира) — удобнее обрабатывать в коде, подписавшись на **QuestCompleted** и вызвав свою логику выдачи наград.
- **Ветвление по выбору в диалоге** — зачёт целей и открытие следующих квестов можно вызывать из UnityEvent по кнопкам ответов, но связку «выбор A → квест 1, выбор B → квест 2» проще держать в скрипте.
- **Проверка доступности квеста для UI** (показывать квест только если выполнены StartConditions) — делается из кода через повторную оценку условий или через свой флаг, т.к. QuestManager не предоставляет «проверить без принятия» через UnityEvent.
