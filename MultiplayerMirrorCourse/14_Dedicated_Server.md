---

## Цели

Собрать серверный билд без UI, читать аргументы командной строки, стартовать `NetworkManager` в headless-режиме.

---

## Проблема и контекст

| Host-only продакшн | Риск |
|--------------------|------|
| Игрок = сервер | Чит на машине хоста, исчезновение матча при выходе |
| Тяжёлый UI на сервере | Память и CPU впустую |

---

## Теория

**Server build** определяет `UNITY_SERVER`: отключайте камеры, пост-обработку, часть аудио и UI через условную компиляцию.

Порты задаются на транспорте (`PortTransport`). В v96 используйте флаг **`listen`** вместо устаревшего `dontListen` при чтении документации сторонних гайдов.

Примеры Mirror обновляют **Edgegap plugin** — смотрите актуальную версию в репозитории примеров.

---

## Практика

```csharp
void Awake()
{
#if UNITY_SERVER
    Destroy(uiRoot);
#endif
}
```

Чтение `-port 7777` из `Environment.GetCommandLineArgs()`, присвоение порта транспорту, затем `StartServer()`.

---

## Типичные ошибки

- `Update` в UI-скриптах всё ещё крутится, если объект не уничтожен — уничтожайте корень UI.
- Забыли **Linux** build target для VPS.

---

## Советы и паттерны

- `Application.targetFrameRate` под тикрейт симуляции.
- Логи в файл + ротация (см. урок 15 / будущий ops-гайд).
