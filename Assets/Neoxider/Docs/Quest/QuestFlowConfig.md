# QuestFlowConfig

**Что это:** `QuestFlowConfig` — ScriptableObject для организации квестов по потокам: последовательные цепочки (strict order) и отдельные standalone-квесты.

**Зачем нужен:** чтобы явно разделить:
- сюжетные/линейные квесты (принимать только после завершения предыдущего),
- независимые побочные квесты.

---

## Поля

| Поле | Описание |
|------|----------|
| **Chains** | Список цепочек `QuestChain`. |
| **Standalone Quests** | Список независимых квестов вне цепочек. |

### QuestChain

| Поле | Описание |
|------|----------|
| **Chain Id** | Технический id цепочки. |
| **Display Name** | Имя цепочки для UI/редактора. |
| **Strict Order** | Если включено, квест `N` можно принять только после `Completed` у квеста `N-1`. |
| **Quests** | Порядок квестов внутри цепочки. |

---

## Как использовать

1. Создать ассет: **Create → Neoxider → Quest → Quest Flow Config**.
2. Добавить цепочки и/или standalone квесты.
3. В вашей вьюшке/логике перед `AcceptQuest(...)` проверять доступность через `QuestFlowConfig.CanAcceptQuest(...)`.

---

## Поведение по умолчанию

- Квест, который не найден ни в одной strict-цепочке, считается доступным (если не принят ранее и выполнены Start Conditions в `QuestManager`).
- Первый квест strict-цепочки доступен сразу.
- Остальные квесты strict-цепочки блокируются до `Completed` предыдущего.
