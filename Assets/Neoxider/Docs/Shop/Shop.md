# Shop

**Назначение:** Главный контроллер внутриигрового магазина. Отвечает за генерацию товаров (UI) на основе `ShopItemData`, обработку покупок (взаимодействие с балансом игрока), сохранение купленных товаров и управление выбранным (экипированным) предметом.

## Подключение

1. Добавьте компонент через меню `Add Component > Neoxider > Shop > Shop` на пустой объект на сцене.
2. В поле `_shopItemDatas` добавьте список товаров (`ScriptableObject`).
3. Назначьте `_prefab` (префаб товара с компонентом `ShopItem`) и `_container` (обычно это Layout Group, куда будут спавниться кнопки).
4. Если вы используете окно превью, назначьте `_shopItemPreview`.

## Основные настройки (Inspector)

| Поле | Описание |
|------|----------|
| `_prices` | Массив цен по умолчанию (если `ShopItemData` не используются). |
| `_shopItemDatas` | Массив данных товаров (ScriptableObject `ShopItemData`), откуда берутся иконки, цены и описания. |
| `_shopItemPreview` | Ссылка на UI-компонент предпросмотра (тот же класс `ShopItem`), который будет отображать выбранный товар перед покупкой. |
| `_shopItems` | (Заполняется автоматически) Массив UI-товаров, уже находящихся на сцене. |
| `_useSetItem` | Если `true`, магазин отслеживает "Экипированный/Выбранный" предмет, выделяя его среди остальных. |
| `_autoSubscribe` | Если `true`, магазин сам подпишется на кнопки `buttonBuy` у всех `ShopItem`. |
| `_activateSavedEquipped` | Автоматически выбирать (Select) сохраненный экипированный предмет при загрузке сцены. |
| `_keySave` | Базовый ключ для сохранения прогресса покупок в `SaveProvider`. |
| `_keySaveEquipped` | Ключ для сохранения ID экипированного предмета. |
| `_changePreviewOnPurchaseFailed` | Менять ли превью на предмет, если при попытке покупки не хватило денег. |
| `_container` | `Transform`, внутрь которого будут спавниться префабы товаров. |
| `_prefab` | Префаб товара (компонент `ShopItem`). |
| `moneySpendSource` | Источник списания денег. Объект должен реализовывать интерфейс `IMoneySpend`. Если оставить пустым, используется глобальный `Money.I`. |

## События (Events)
- `OnSelect` — Вызывается, когда предмет выбран (экипирован).
- `OnPurchased` — Вызывается при успешной покупке.
- `OnPurchaseFailed` — Вызывается, когда денег на покупку не хватило.
- `OnLoad` — Вызывается после загрузки данных магазина.
