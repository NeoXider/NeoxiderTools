## Что должно получиться

Вы понимаете, почему подключение к `localhost` не доказывает работу игры в интернете. Для своей игры вы выбираете прямой host, relay или dedicated server.

---

## Проблема

Игроки обычно находятся за роутером, firewall или CGNAT. Другой клиент не может просто подключиться к их машине, если нет публичного адреса и проброшенного порта.

---

## Варианты подключения

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| Direct host + port forwarding | Дёшево для прототипа | Сложно для игроков, CGNAT ломает. |
| Relay | Проще подключение | Стоимость, задержка, зависимость от провайдера. |
| Dedicated server | Контроль, честность, логи | Инфраструктура и стоимость. |
| Steam/EOS platform path | Удобно в экосистеме | Vendor lock-in, платформенные ограничения. |

---

## Практика

1. Узнайте transport и протокол: UDP/TCP/WebSocket.
2. Запустите server не через `localhost`, а по LAN IP.
3. Подключитесь с другой машины в локальной сети.
4. Проверьте внешний сценарий: другая сеть или VPS.
5. Зафиксируйте, где нужен port forwarding.

---

## Проверка себя

- Вы знаете, какой порт и протокол открыть.
- Подключение проверено не только на одной машине.
- Для мобильных/домашних игроков есть план при CGNAT.
- В UI есть понятная ошибка "сервер недоступен".

---

## Минимальная диагностика

| Симптом | Что проверить |
|---------|---------------|
| Работает LAN, не работает интернет | Public IP, port forwarding, firewall, CGNAT. |
| Port открыт, connect нет | Protocol mismatch: UDP/TCP/WebSocket. |
| Client видит wrong address | Backend/lobby отдаёт internal IP вместо public endpoint. |
| Игроки не могут host | Нужен relay/dedicated или platform P2P. |

---

## Частые ошибки

- "У меня работает localhost" считается интернет-тестом.
- Открыт TCP, а transport использует UDP.
- Сервер слушает только localhost.
- Игрокам предлагают вручную настраивать роутер без альтернативы.

---

## Лайфхаки

- Для публичного теста чаще выбирайте dedicated или relay.
- Direct connect подходит для dev, LAN, технической аудитории.
- Документируйте network path в `NETWORKING_DECISIONS.md`.

---

## Профессиональный минимум

- UI объясняет connection failure без технической каши.
- Network path выбран под аудиторию, а не под удобство разработчика.
- Port/protocol записаны в docs и hosting config.
- CGNAT не считается edge case для публичной игры.
